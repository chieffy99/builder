# การ Map คำศัพท์จากเอกสาร Anti-Normalization

จากการวิเคราะห์เอกสารและคำขอของคุณ ผมจะนำเสนอการ Map คำศัพท์หลักในระบบ Anti-Normalization ตามที่คุณต้องการ โดยไม่มีความซับซ้อนเกินจำเป็น:

## Core Dictionary Mapping

| คำศัพท์ | ความหมาย | บทบาทในระบบ |
|---------|----------|-------------|
| **Sting** | หน่วยธุรกรรมที่สมบูรณ์ | เปรียบเสมือนแถว (row) ในฐานข้อมูล แต่ละ Sting เป็นเหตุการณ์/เรื่องราวที่สมบูรณ์ |
| **Event** | Narrative/Sting/Sentence | หมายถึงแถวข้อมูลที่บันทึกเหตุการณ์หนึ่งๆ ครบถ้วน |
| **Word** | Rule/Field/Data dict | หมายถึงคอลัมน์ที่กำกับด้วยกฎเกณฑ์ (ไม่ใช่แค่เก็บค่า) |
| **Logic** | Reasoning/Language/Formula | สูตรคำนวณที่ปรับเปลี่ยนตามแพลตฟอร์ม เช่น SUMIFS |
| **Space** | Dimensions/Position/Slot | โครงสร้างเชิงพื้นที่ที่บังคับการจัดวางข้อมูล |
| **Time** | Rotten/Index/tenses | การลำดับข้อมูลตามเวลาและการเสื่อมสลายของข้อมูล |
| **Gravity** | Relation/ID/SOV | ความสัมพันธ์ระหว่างเอนทิตีในรูปแบบ Subject-Object-Verb |
| **Transform** | Shift/Stat/Germinate | กระบวนการเปลี่ยนแปลงสถานะหรือมุมมองของข้อมูล |
| **StatN** | ตัวเลข 0-9 | กำหนดสถานะเชิงตัวเลข เช่น 0=เงินสด, 1=เงินเชื่อ |
| **StatC** | ตัวอักษร A-Z | กำหนดประเภทของข้อมูล เช่น A=Result, B=Count, C=Static |
| **Slot** | ช่องเก็บค่า 1-n | พื้นที่เก็บข้อมูลที่มีความหมายเฉพาะตามบริบท |
| **ID1** | ผู้กระทำ/เจ้าของ | รหัสระบุตัวผู้กระทำหรือเจ้าของในธุรกรรม |
| **ID2** | ประเภทการกระทำ | รหัสระบุประเภทของการกระทำหรือธุรกรรม |
| **Date** | วันที่ | วันที่เกิดเหตุการณ์หรือธุรกรรม |

## Transaction Type Mapping (ID2)

| รหัส | ความหมาย | จาก (from) | ไปยัง (to) | ผลลัพธ์ |
|------|----------|------------|------------|---------|
| **PO** | สั่งสินค้า (Order) | Sup_id | ST | เพิ่ม |
| **RV** | ตีกลับ (Revert) | Sup_id | ST | ลด |
| **BL** | คงเหลือ (Balance) | ST | ST | เช็ค |
| **PK** | เบิก (Picking) | Dist_ID | ST | ลด |
| **RT** | ส่งคืน (Return) | Dist_ID | ST | เพิ่ม |
| **DM** | เสียหาย (Damage) | Dist_ID | Dist_ID | ลด |
| **RF** | อ้างอิง (Refer) | Dist_ID | Dist_ID | ยกยอด |
| **BYT** | ซื้อ (Buy) | Cust_OT | OT | ลด |
| **BYK** | ซื้อ (Buy) | Cust_MK | MK | ลด |
| **BYH** | ซื้อ (Buy) | Cust_SH | SH | ลด |

## Entity Type Mapping (ID1)

| รหัส | ความหมาย | ตัวอย่าง |
|------|----------|---------|
| **Per_sup** | ซัพพลายเออร์ | SU, PS, SK |
| **ST_ID** | คลังสินค้า | ST |
| **DistRv** | ศูนย์กระจายสินค้า | OT, MK, SH |
| **Cust_OT** | ลูกค้าของ OT | OTE, OT00, OT01, ..., OTn |
| **Cust_MK** | ลูกค้าของ MK | MKE, MK00, MK01, ..., MKn |
| **Cust_SH** | ลูกค้าของ SH | SHE, SH00, SH01, ..., SHn |

## Status Code Mapping

| StatN | ความหมาย |
|-------|----------|
| **0** | เงินสด |
| **1** | หนี้, เงินเชื่อ |
| **2** | ชำระสินเชื่อ |
| **3** | เงินโอน |

| StatC | ความหมาย |
|-------|----------|
| **A** | Result value (ค่าที่ผ่านการคำนวณ) |
| **B** | Count value (ค่าจำนวนที่ยังไม่ผ่านการคำนวณ) |
| **C** | Static value (ค่าคงที่) |

## Slot Function Mapping

| Slot | ชื่อเฉพาะ | บทบาทหน้าที่ |
|------|----------|-------------|
| **slot1-slot5** | หลัก | ค่าหลักสำหรับการคำนวณตามบริบท |
| **slot6** | reduce | ค่าส่วนลดหรือการลดมูลค่า |
| **slot7** | deduce | ค่าสำหรับการหักหรือตัดออก |

## Formula Mapping

| บริบท | Formula | ความหมาย |
|-------|---------|----------|
| รวมยอดขาย | `SUMIFS(A0slot1, A0ID1,"DistRv", A0ID2,"SA")` | รวมยอดขายทั้งหมดของศูนย์กระจายสินค้า |
| ยอดสินค้าคงเหลือ | `SUMIFS(A0slot1, A0ID1,"ST_ID", A0ID2,"BL")` | ตรวจสอบยอดสินค้าคงเหลือในคลัง |
| ยอดหนี้ | `SUMIFS(A0slot1, A0StatN,"1")` | รวมยอดหนี้ทั้งหมด (StatN=1) |
| ยอดเงินสด | `SUMIFS(A0slot1, A0StatN,"0")` | รวมยอดเงินสดทั้งหมด (StatN=0) |

## ตัวอย่างการใช้งาน

| บริบท | Sting (แบบเต็ม) |
|-------|-----------------|
| ซื้อสินค้า | `2025/06/05, Cust_OT, BYT, 0, B, 1500, -, -, -, -, -, -` |
| ขายสินค้า | `2025/06/05, DistRv, SA, 0, B, 2000, -, -, -, -, 200, -` |
| รับสินเชื่อ | `2025/06/05, Per_sup, PO, 1, B, 10000, -, -, -, -, -, -` |
| ชำระหนี้ | `2025/06/05, Per_sup, PO, 2, A, 5000, -, -, -, -, -, -` |

## ตัวอย่างการแปลงประโยคเป็น Sting

| ประโยคธรรมชาติ | Sting Equivalent |
|----------------|-------------------|
| "วันนี้ นาย A ไปธนาคาร สาขา... เพื่อเปิดบัญชี และรับเงินสินเชื่อ" | `2025/06/05, A, BANK, 1, B, 50000, -, -, -, -, -, -` |
| "ธนาคารอนุมัติคำขอสินเชื่อของ นาย A เมื่ออาทิตย์ที่แล้ว" | `2025/05/29, BANK, LOAN, 2, A, 50000, -, -, -, -, -, A` |

## ความสัมพันธ์ระหว่างองค์ประกอบ OntologyCore

```
Event (Row/Sting) ─┐
                   │
Word (Column) ─────┼─── ทุกองค์ประกอบทำงานร่วมกัน
                   │    เพื่อสร้างระบบข้อมูลที่สมบูรณ์
Logic (Formula) ───┤
                   │
Space (Slot) ──────┤
                   │
Time (Date) ───────┤
                   │
Gravity (ID) ──────┤
                   │
Transform (Stat) ──┘
```

นี่คือการ Map คำศัพท์หลักในระบบ Anti-Normalization ตามที่คุณขอ โดยไม่มีความซับซ้อนเกินจำเป็น ผมหวังว่าการแสดงผลในรูปแบบตารางและการจัดหมวดหมู่ที่ชัดเจนจะช่วยให้คุณเข้าใจระบบได้ดียิ่งขึ้น
